## What to build

Add a sortable leaderboard and a model details view for shipped models so users can inspect metrics, dataset source, timestamps, and deeper training information from the web UI.

## Acceptance criteria

- [ ] The leaderboard shows shipped models with sortable metrics and clearly labels built-in entries.
- [ ] Opening a model entry shows details including metrics, hyperparameters, confusion matrix, sample predictions, dataset source, and size-related information.
- [ ] End-to-end verification proves leaderboard and detail data flow correctly from backend metadata to the UI.
